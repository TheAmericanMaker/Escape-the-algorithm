# Troubleshooting & FAQ

Having trouble? This page covers the most common issues. If you don't find your answer here, [open an issue](https://github.com/TheAmericanMaker/Escape-the-algorithm/issues) — it helps us improve the docs for everyone.

---

## Installation problems

### `command not found: eta` after installing

The `eta` command isn't on your system's PATH. A few things to try:

**If you used `pip`:**
```bash
pip install --user escape-the-algorithm
```
Then restart your terminal and try again. If it still doesn't work, use:
```bash
python -m eta
```

**Recommended fix:** install with `pipx` instead, which handles PATH automatically:
```bash
pip install pipx
pipx install escape-the-algorithm
```

---

### `command not found: pip`

Try `pip3` instead:
```bash
pip3 install escape-the-algorithm
```

If `pip3` also fails, Python may not be installed. Download it from [python.org](https://www.python.org/downloads/) — choose the latest Python 3 version and check "Add Python to PATH" during installation on Windows.

---

### `ERROR: Could not find a version that satisfies the requirement`

This usually means your Python version is too old. `eta` requires Python 3.9 or newer.

Check your version:
```bash
python --version
```

If it shows Python 2.x or Python 3.8 or older, update Python from [python.org](https://www.python.org/downloads/).

---

### Python is installed but `pip install` still fails

On some systems (Debian/Ubuntu), you may see an error about "externally managed environment." Fix it with:
```bash
pip install --break-system-packages escape-the-algorithm
```
Or use `pipx` as described above, which avoids this entirely.

## Running eta

### `File not found` error

This means eta can't find your export file in the current folder.

**Solution:** navigate to the folder where your file is before running eta:
```bash
# On Windows
cd Downloads

# On macOS or Linux
cd ~/Downloads
```

Then list the files to confirm your export file is there:
```bash
# Windows
dir

# macOS / Linux
ls
```

Copy the exact filename (including extension) and run eta again:
```bash
eta convert your-exact-filename.csv
```

---

### `Could not detect platform format`

eta looks at the file contents to figure out which platform it came from, but couldn't identify this one.

**Solutions:**
- Make sure you downloaded the right file. See the platform-specific guides: [YouTube](youtube.md) | [Reddit](reddit.md) | [Twitter/X](twitter.md) | [TikTok](tiktok.md) | [Spotify](spotify.md)
- Name the platform explicitly rather than using `convert`:
  ```bash
  eta youtube subscriptions.csv
  eta reddit subscribed_subreddits.csv
  ```
- If the file is compressed (`.zip`, `.tar.gz`), extract it first — eta reads the files inside, not the archive.

---

### `No subscriptions found`

The file was read successfully but contained no data. Common causes:

- You downloaded the wrong file from the platform's export. Check the guide for your platform.
- The file is for a different account than the one with subscriptions.
- The export is for a newly created account with no subscriptions yet.

---

### eta runs but the output OPML is empty (0 feeds)

Same as above — the parser read the file but found no subscriptions. Double-check the [platform guide](getting-started.md) to make sure you're using the right export file and the right `eta` command.

## Platform-specific issues

### YouTube: `eta youtube` gives 0 results

Make sure you're using the file from Google Takeout, not YouTube Studio. The correct file is named `subscriptions.csv` and is inside the `YouTube and YouTube Music/subscriptions/` folder of your Takeout archive.

See [YouTube guide](youtube.md) for step-by-step instructions.

---

### Reddit: my subreddits aren't showing up

Reddit changed its export format over time. `eta` supports:
- The GDPR CSV export (a file named `subscribed_subreddits.csv`)
- A plain text file with one subreddit per line (e.g. `programming`, `r/programming`, or the full URL)

If you have a different format, [open an issue](https://github.com/TheAmericanMaker/Escape-the-algorithm/issues) with a sample (remove any personal info).

---

### Twitter/X: the OPML imports but feeds show no content

This is a known limitation. Twitter/X blocked public RSS access in 2023. The feeds generated by `eta` rely on [Nitter](https://github.com/zedeus/nitter), a community-maintained Twitter front-end. Public Nitter instances may be unreliable.

For reliable Twitter RSS, you'd need to self-host a Nitter instance. See [Twitter/X guide](twitter.md) for details.

---

### Spotify: most podcasts show `no rss`

Spotify's export doesn't include RSS URLs. Run with `--resolve-rss` to look them up automatically via the Podcast Index API:
```bash
eta spotify Follow.json --resolve-rss
```
This requires free API credentials. See [Spotify guide](spotify.md) for setup instructions.

## RSS reader import

### My RSS reader says the OPML file is invalid

The OPML file may have been generated with an older version of `eta`. Try regenerating it:
```bash
eta convert your-export-file --verbose
```

If the problem persists, [open an issue](https://github.com/TheAmericanMaker/Escape-the-algorithm/issues) and attach the OPML file.

---

### I imported the OPML but some feeds show errors

Some channels may have changed their RSS URL, or the platform-specific proxy (Nitter, ProxiTok) may be having issues. This is outside eta's control — eta generates the feed URLs, but the feeds themselves are served by the platform or community proxy.
